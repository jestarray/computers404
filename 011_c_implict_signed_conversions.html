C Implicit Signed Conversions

In C, operations on numbers that mix signed and unsigned have a weird unwanted surprising trick, and that is: signed values implicitly cast to unsigned values. Such operations include comparison operations as well as the 4 basic arithmetic operations(+, -, *, /), and comparisons(>=, ==, etc). This can result into some very hard to spot bugs.

By default numbers are signed. Numbers that have a “U” next to them means unsigned literal, e.g 264U.

There are compiler flags and static analysis tools you can use to warn you when this happens. More newer languages(rust) will force you to cast explicitly.
